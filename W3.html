Capstone Project Week 3


Team work

This week, we kept brainstorming game ideas, coding standards about variable naming, and how to keep the code clean. We asked our mentors for advice on how to push our limits,while having a practical expectation. Our mentors provided us with a reference “PLATEUP!” to give us ideas on adding more features to our game. We also reviewed each team member’s TDD file.

My work

This week, I built the blog page on my personal website. I wanted to keep the website header clean, so I decided not to create a new item on the header. Therefore, I put the blog page under the portfolio.


Then, I worked on improving the localization system. I practiced using this feature as my TDD file topics. I learned to use the UML diagram and flow chart, and made sure the chart I made can be understood by the readers even if they have no clue what our game is like. Our team and mentors encouraged me to have the localization system done in a centralized way because it is more organized. My draft was working, but the structure of coding was not clean. I received suggestions from my professor James to improve the functionality of the localization system, which made the code more readable.

The next challenge I had was to push the update to GitHub since this system has impacts on the menu scene including the Canvas, buttons, and toggles settings. I need to discuss it with the team members who were working on it to avoid a merge conflict. I think it is one of the most valuable parts of this course: not just making a game but also working in a team in game development.
